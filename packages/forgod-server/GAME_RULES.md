# For God 규칙서 (구현 기준)

이 문서는 `packages/forgod-server`의 현재 구현(`src/game/*`)과 `CLAUDE.md`를 기준으로 정리한 규칙서다.
기획 규칙과 구현 규칙이 다를 때는 **현재 코드 동작**을 우선으로 적었다.

## 1. 게임 개요

- 플레이어는 용사가 되어 신성(천사) 또는 타락(마왕) 방향의 목표를 수행한다.
- 핵심 자원:
  - 체력(HP)
  - 능력치 주사위: 힘/민첩/지능 (각 2개)
  - 몬스터 정수(`monsterEssence`)
  - 점수: 신앙 점수(`faithScore`), 마왕 점수(`devilScore`)
  - 타락 주사위(`corruptDice`)

## 2. 직업과 상태

- 직업(`HeroClass`):
  - `warrior`(전사)
  - `rogue`(도적)
  - `mage`(법사)
- 상태(`HeroState`):
  - `holy`(신성)
  - `corrupt`(타락)

상태 핵심 차이:
- 타락 용사는 기본적으로 신전에 진입할 수 없다. 단, 마검 보유 시 가능.
- 타락 용사는 화염 타일 피해를 받지 않는다.
- 타락 용사는 마검 뽑기 가능.

## 3. 능력치, 레벨, 체력

- 초기 능력치: 힘/민첩/지능 모두 `[1, 1]`.
- 레벨은 `max(힘합, 민첩합, 지능합)`.
- 직업/레벨별 최대 체력 테이블을 사용한다 (`constants.ts`의 `HEALTH_BY_CLASS_AND_LEVEL`).
- 능력치 상승으로 최대 체력이 오르면 현재 체력도 같은 수치만큼 오른다.

## 4. 보드와 지형

타일 타입:
- `plain`, `village`, `mountain`, `lake`, `hill`, `swamp`, `fire`, `temple`, `castle`, `monster`

이동 비용(`TERRAIN_MOVEMENT_COST`):
- 평지/마을/화염/신전/마왕성: 3
- 늪: 5
- 언덕: 남은 이동력 전부 소모(`all`)
- 산/호수/몬스터 타일: 이동 불가(`blocked`)

추가 규칙:
- 언덕은 진입 시점에 남은 이동력이 3 이상이어야 하며, 진입하면 이동력이 0이 되고 행동 단계로 즉시 전환된다.
- 다른 생존 플레이어가 있는 칸으로 이동할 수 없다.
- 산/호수에 갇혀 턴 시작 시 이동 주사위를 굴리면 인접 이동 가능 타일로 강제 탈출 처리된다.

## 5. 게임 시작

- 플레이어 초기 상태:
  - 신성(`holy`)
  - 타락 주사위 없음
  - 몬스터 정수 0
  - 점수 0
- 시작 위치:
  - 직업별 지정 시작 칸 배열(`STARTING_POSITIONS`)에서 순서대로 배치
- 초기 턴 순서:
  - 도적이 있으면 첫 도적이 선턴
  - 나머지 플레이어
  - 마지막 `monster`

## 6. 턴 구조

플레이어 턴은 `move` → `action` 2단계다.

### 6.1 이동 단계 (`move`)

가능 액션:
- `ROLL_MOVE_DICE`
- `MOVE`
- `END_MOVE_PHASE`

`ROLL_MOVE_DICE` 처리:
- 2d6 + 민첩 합 = 이동력
- 턴 시작 자동 처리:
  - 산/호수 탈출 처리
  - 속박(`isBound`) 해제 + 이번 턴 이동력 0
  - 무적 태세(`ironStanceActive`) 해제
  - 화염 타일 위면 피해 10(타락이 아니고 화염 비활성 버프가 없을 때)

### 6.2 행동 단계 (`action`)

가능 액션:
- 기본 공격 `BASIC_ATTACK` (턴당 1회)
- 스킬 사용 `USE_SKILL`
- 능력치 주사위 굴리기 `ROLL_STAT_DICE`
- 턴 종료 `END_TURN`

턴 종료 시:
- 남은 이동력은 `leftoverMovement`로 저장
- `usedSkillCost`, `hasUsedBasicAttack` 리셋
- 마을이면 체력 회복
  - 자기 직업 마을: +10
  - 타직업 마을: +5

라운드 종료 후:
- 몬스터 턴 실행
- 사망자 부활 카운트 감소
- 0이 되면 부활: 해당 직업 시작 위치(첫 칸), 체력 절반
- 부활자는 천사/마왕 계시 1장 랜덤 드로우
- 스킬 쿨다운 1 감소
- 다음 라운드 턴 순서: `leftoverMovement` 높은 순 + `monster`

## 7. 전투 규칙

### 7.1 기본 공격

- 인접 대상(거리 1)만 공격 가능.
- 공격력 기본값: 힘 합.
- `poisonActive`면 민첩 합 추가(사용 후 소모).
- 공격자는 공격 시 은신 해제.

대상별 제약:
- 은신 중 대상은 기본 공격 대상 불가.
- 마검 소유자는 몬스터를 기본 공격할 수 없다.
- 몬스터도 마검 소유자를 공격하지 않는다.

피해 경감:
- 무적 태세(`ironStanceActive`)면 받는 피해에서 힘 합만큼 감소.

처치/타락:
- 신성 용사가 신성 용사를 처치하면 공격자 자동 타락 + 타락 주사위 1.
- 타락 용사가 누군가 처치하면 타락 주사위 +1(최대 6).
- 신성 용사가 타락 용사를 처치한 경우는 즉시 자동 타락이 아니라 `CHOOSE_CORRUPTION` 선택 액션 대상.

### 7.2 몬스터 정수

- 몬스터에게 준 실제 피해량(과잉 제외)만큼 `monsterEssence` 획득.
- 몬스터 제물 분배 로직은 TODO 상태이며 현재 미구현.

## 8. 스킬 규칙

공통:
- 자기 직업 스킬만 가능.
- 쿨다운 0이어야 사용 가능.
- 턴 중 스킬 총 비용(`usedSkillCost`) + 이번 스킬 비용 <= 지능 합.
- 사용 시 스킬 쿨다운과 비용이 즉시 반영.

### 8.1 전사

- `warrior-charge` 돌진: 거리 2 대상의 인접 칸으로 한 칸 이동, 쿨다운 중인 나머지 스킬 하나의 쿨다운을 지정해서 1 감소.
- `warrior-power-strike` 일격 필살: 다음 기본 공격의 피해를 힘x2로 강화.
- `warrior-throw` 던져버리기: 인접 플레이어를 2칸 내 위치로 던짐. 산에 던지면 대신 대상에게 힘 피해 발생.
- `warrior-iron-stance` 무적 태세: 다음 턴 시작 전까지 자신의 힘 수치 만큼 받는 피해 감소.
- `warrior-sword-wave` 검기: 지정 방향 직선 3칸 타일에게 힘 피해.

### 8.2 도적

- `rogue-poison` 독 바르기: 다음 기본 공격에 민첩 피해 추가.
- `rogue-shadow-trap` 그림자 함정: 현재 칸 함정 설치(최대 3개, 초과 시 기존 함정 중 1개 제거).
- `rogue-backstab` 배후 일격: 대상 뒤 칸으로 이동 후 민첩 피해.
- `rogue-stealth` 은신: 대상 지정 피해 면역, 범위 피해를 받거나 공격하면 해제.
- `rogue-shuriken` 표창: 직선 첫 대상에게 민첩 피해, 2칸 이상이면 민첩 * 2 피해. 산에 막힘.

### 8.3 법사

- `mage-enhance` 강화: 다음 1회 스킬 강화 상태 부여.
- `mage-magic-arrow` 마법 화살:
  - 플레이어 대상 사거리 2
  - 몬스터 대상 사거리 1
  - 지능 피해를 줌
  - 강화 시 분신이 마법 화살 공격
- `mage-clone` 분신:
  - 기본: 분신 생성(플레이어당 1개 유지)
  - 강화 상태 + 기존 분신 존재 시: 분신과 위치 교환
- `mage-burst` 방출: 주변 1칸 모두에게 지능 피해. 강화 시 피해 받은 대상 한 칸 씩 뒤로 밀림.
- `mage-meteor` 메테오: 지정 칸 지능 피해, 강화 시 2배.
  - `meteorImmune` 버프가 있으면 몬스터는 메테오 피해 무시.

## 9. 함정, 속박, 화염

- 함정:
  - 다른 플레이어 함정 칸 진입 시, 설치자 민첩 합만큼 피해.
  - 발동한 함정은 제거.
  - 설치자는 은신 상태가 된다.
- 속박(`isBound`):
  - 턴 시작 `ROLL_MOVE_DICE` 시 해제되며 그 턴 이동력 0.
- 화염:
  - 비타락 플레이어는 화염 피해 10.
  - 발록 사망 중에는 화염 비활성(`fireTileDisabled`).

## 10. 몬스터 페이즈

- 몬스터 주사위 6개를 굴리고 오름차순 정렬.
- 각 몬스터는 자기 `diceIndices`에 해당하는 합으로 행동.
- "효과"와 "공격"은 분리되어 둘 다 실행된다.

공통 공격:
- 타겟 타일 인덱스 = `diceSum % adjacentTiles.length`
- 해당 타일에 인접 생존 플레이어가 있으면 `diceSum` 피해.

예외:
- 트롤: 대상이 타락이면 합이 홀수일 때만 공격.
- 은신 대상 공격 스킵.
- 마검 소유자 공격 스킵.

고유 효과:
- 하피: 합>=7, 인접 용사를 몬스터 반대 방향으로 1칸 밀침.
- 그린딜로: 합>=7로 공격 성공 시 대상 속박.
- 리치: 합>=15, 다음 라운드까지 몬스터 메테오 면역.
- 히드라: 합>=15, 잃은 체력의 2배 회복(최대 체력 초과 가능).
- 골렘: 합==12, 다음 라운드까지 기본 공격 면역.
- 발록: 사망 시 화염 비활성, 이후 몬스터 페이즈에 부활하며 화염 재활성.

## 11. 계시 시스템

### 11.1 계시 카드

- 천사/마왕 계시 덱(`REVELATIONS`)에서 드로우.
- 완료 시:
  - 점수/타락점수/추가 드로우 보상 적용
  - `revelations`에서 `completedRevelations`로 이동
  - `isGameEnd` 계시면 `GAME_OVER` 이벤트 추가

### 11.2 완료 조건 분류

직접 체크형:
- 위치/점수/상태 기반 조건(`checkRevelationCondition`)

이벤트 자동완료형:
- 전투 컨텍스트로 자동 체크(`processEventRevelations`)
- 예: 발록 공격/처단, 선제공격, 마을 습격, 타락 유혹

특수:
- `angel-7` 천사의 가호:
  - 타락 용사의 치명타를 받으면 체력 1로 생존
  - 즉시 계시 완료 및 보상 적용

주의:
- 제물 시스템은 TODO가 많아 일부 계시는 임시 조건(위치 등)으로 처리된다.

## 12. 마검 규칙

`DRAW_DEMON_SWORD` 조건:
- 플레이어가 타락 상태
- 마왕성 방문으로 마검 위치 인지(`knowsDemonSwordPosition=true`)
- 현재 위치가 마검 위치
- 마검이 아직 미획득(`demonSwordPosition !== null`)

획득 시:
- `hasDemonSword=true`
- `demonSwordPosition=null`

## 13. 승리 조건

### 13.1 마왕 승리 트리거

- 타락 상태 + 마검 보유 + 신전 진입.
- 실제 승자: `(devilScore - faithScore)`가 가장 높은 플레이어.

### 13.2 천사 승리 트리거

- 신성 상태 + 신앙 점수 5 이상 + 마왕성 진입.
- 실제 승자: `(faithScore - devilScore)`가 가장 높은 플레이어.

### 13.3 계시 승리

- `isGameEnd=true` 계시를 완료한 플레이어 즉시 승리.

## 14. 현재 구현 주의사항 (기획과 다를 수 있음)

- 인원 제한(`maxPlayers`)은 설정값이 있으나 `createGame`에서 강제 검증하지 않는다.
- 계시 중 일부는 TODO 상태(특히 제물/교환 기반 조건).
- 산/호수 탈출 시 다중 후보가 있어도 현재는 자동으로 첫 번째 칸을 선택한다.
- 타락 해제(`CHOOSE_HOLY`)는 코드상 "부활 직후 전용" 강제가 아니라 `corrupt` 상태면 가능하다.
