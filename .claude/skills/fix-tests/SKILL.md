# /fix-tests

테스트를 실행하고 모든 테스트가 통과할 때까지 코드를 수정합니다.

## 대상 패키지: $ARGUMENTS

인자에 따라 실행할 테스트 명령어:
- `forgod` → `pnpm forgod:test`
- 인자가 없으면 사용자에게 패키지를 선택하도록 요청

## 실행 절차

1. 위 명령어로 테스트를 실행합니다.
2. 실패한 테스트가 있으면:
   - 실패 메시지를 분석합니다.
   - 관련 소스 코드를 읽고 문제를 파악합니다.
   - 테스트 코드가 잘못된 경우 테스트를 수정합니다.
   - 소스 코드가 잘못된 경우 소스 코드를 수정합니다.
   - 다시 테스트를 실행합니다.
3. 모든 테스트가 통과하면 완료입니다.

## 규칙

- 테스트의 의도를 파악하고, 테스트가 검증하려는 동작이 올바른지 먼저 확인합니다.
- 소스 코드의 버그인 경우 소스 코드를 수정합니다.
- 테스트 코드가 잘못된 기대값을 가진 경우에만 테스트를 수정합니다.
- 수정 후 반드시 테스트를 다시 실행하여 확인합니다.
- 한 번에 하나의 실패만 수정하고 바로 재실행합니다.
