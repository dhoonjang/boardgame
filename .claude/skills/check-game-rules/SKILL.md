# /check-game-rules

CLAUDE.md의 게임 규칙과 실제 구현이 일치하는지 감사합니다.

## 대상 게임: $ARGUMENTS

인자가 없으면 사용자에게 게임을 선택하도록 요청합니다.

## 워크플로우

### 1단계: CLAUDE.md 읽기
- `packages/<game>-core/CLAUDE.md`를 읽어 문서화된 규칙 목록을 정리합니다.

### 2단계: 구현 코드 분석
다음 파일들을 읽고 규칙별로 구현 여부를 확인합니다:
- `packages/<game>-core/src/types.ts` - 타입이 문서와 일치하는지
- `packages/<game>-core/src/constants.ts` - 상수가 문서와 일치하는지
- `packages/<game>-core/src/engine/` - 엔진 로직이 규칙과 일치하는지

### 3단계: 테스트 커버리지 확인
- `packages/<game>-core/src/__tests__/` - 각 규칙에 대한 테스트가 있는지

### 4단계: 감사 보고서 출력
각 규칙에 대해 다음 상태를 보고합니다:

| 상태 | 의미 |
|------|------|
| OK | 문서와 구현이 일치 |
| MISMATCH | 문서와 구현이 다름 (상세 내용 포함) |
| NOT_IMPLEMENTED | 문서에는 있으나 미구현 (TODO 포함) |
| UNDOCUMENTED | 구현은 있으나 문서에 없음 |
| NO_TEST | 구현은 있으나 테스트가 없음 |

### 5단계: 권장 조치
- 불일치 항목별 수정 방향을 제안합니다.
- 사용자에게 문서/코드 중 어느 쪽이 정확한지 확인합니다.

## 검사 항목 (forgod 기준)

### 타입 검사
- [ ] GameState 필드가 문서와 일치하는가
- [ ] Player 필드가 문서와 일치하는가
- [ ] Monster 필드가 문서와 일치하는가
- [ ] GameAction 종류가 문서와 일치하는가

### 상수 검사
- [ ] 최대 체력 테이블이 문서와 일치하는가
- [ ] 지형 이동 비용이 문서와 일치하는가
- [ ] 몬스터 스탯이 문서와 일치하는가
- [ ] 스킬 정보가 문서와 일치하는가

### 로직 검사
- [ ] 능력치 업그레이드 규칙이 문서와 일치하는가
- [ ] 체력 변동 규칙이 문서와 일치하는가
- [ ] 턴 순서 규칙이 문서와 일치하는가
- [ ] 타락 시스템이 문서와 일치하는가
- [ ] 부활 시스템이 문서와 일치하는가
- [ ] 승리 조건이 문서와 일치하는가
- [ ] 마검 시스템이 문서와 일치하는가

## 규칙

- 이 skill은 **읽기 전용**입니다. 코드를 수정하지 않습니다.
- 불일치를 발견하면 사용자에게 보고만 합니다.
- 실제 수정은 `/implement-rule`이나 수동으로 진행합니다.
